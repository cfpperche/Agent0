# 166 - agent0-governance-doctrine - notes

## Design decisions

### 2026-06-07 - parent - Doctrine before mechanism

The user accepted the direction to expand Agent0 as template/governance rather than product. This spec intentionally ships doctrine first and leaves hard gates, gate algebra, and security-lane mechanics as candidate follow-ups.

### 2026-06-07 - parent - Layered taxonomy

The flat category list from the brainstorm was revised after Claude's second-opinion critique: continuous evolution is the temporal spine; quality and security are domains; context and replication are substrate; multi-runtime is transversal; scope admission is meta-governance.

## Deviations

None.

## Tradeoffs

### 2026-06-07 - parent - Entrypoint pointer included

Adding a short pointer in both root entrypoints increases the touched surface, but it makes the doctrine visible before future capacity proposals and keeps Claude/Codex entrypoints aligned.

## Open questions

None for this spec. Follow-up questions are preserved in `spec.md` and should become separate specs only after demand is clear.
