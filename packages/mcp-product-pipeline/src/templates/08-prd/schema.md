# Step 8 — Schema (required sections)

## Required sections

- v1-scope
- success-metric
- acceptance-criteria
- non-goals
- open-questions

## Recommended additional sections

- target-users — recapping the personas with any refinement
- v2-vision — what comes after v1 ships (1-3 bullets)
- timeline — soft launch target if any
- dependencies — external services / APIs / partnerships v1 needs

## Section content guidance

- **v1-scope** — must-haves (5-10 features) with one-line each. Should-haves and nice-to-haves listed separately so the cut is visible.
- **success-metric** — ONE observable metric + target value + measurement window. "Activation rate > 40% in week 1 from cohort" is concrete; "user satisfaction" is not.
- **acceptance-criteria** — BDD scenarios (Given/When/Then) per must-have feature. 2-4 scenarios per feature typically. These are inputs to engineering tests.
- **non-goals** — explicitly out of v1, with one-line reason each. Prevents scope creep during execution.
- **open-questions** — what's not yet decided. Each question has an owner OR a downstream step number where it'll be resolved.
