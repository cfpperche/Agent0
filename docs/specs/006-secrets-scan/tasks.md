## Notes

_Anything that came up during execution that doesn't belong in plan.md but is useful for the PR description or future readers._

- **Don't use `AKIAIOSFODNN7EXAMPLE` as the test-vector key.** Discovered during Brief 2 + Brief 3 implementation: gitleaks 8.21.2 applies a stopword filter to AWS detector matches that contain the token `EXAMPLE`, so the AWS-documented canonical test key is *suppressed* by gitleaks, not *flagged*. This contradicts the intuition that "AWS's documented test key = a gitleaks test vector". For verification scenarios that need to actually trip the AWS detector, use a non-stopworded pattern-valid shape such as `AKIA1234567890ABCDEF` (20 chars, `AKIA` prefix, alphanumeric, no `EXAMPLE`). Safe for fixtures because it's not a real AWS key — just a shape that satisfies the regex.
- The `tool_input.new_string` field name in step 10 may vary by tool variant; check the PostToolUse payload shape for `Edit` (`new_string`), `Write` (`content`), and `MultiEdit` (`edits[].new_string`) before assuming a single accessor.
- Step 6's "no repeated warnings within a single invocation" is per-invocation, not per-session — each `git commit` attempt that hits the no-engine path produces one stderr line. Session-wide dedup would require state in `.claude/.session-state/` and is not worth the complexity for v1.
- **Timing fix delivered in spec 007** (`docs/specs/007-secrets-scan-timing/`). This spec's `PreToolUse(Bash)` hook fires *before* bash executes, so a compound `git add <secret-file> && git commit ...` always scanned an empty index — the staging hadn't happened yet. The fix moves the actual gitleaks call to a native `.githooks/pre-commit` hook (activated per-fork via `git config core.hooksPath .githooks`); this spec's preflight was refactored into a pure shape-gate + override-marker bridge that injects `CLAUDE_SECRETS_OVERRIDE_REASON` for the native layer. Decision values split by layer: preflight emits `skip-not-commit`/`passthrough`/`reject-shape`/`override-pass-through` with `scan_mode: "preflight"`; native emits `allow`/`allow-empty`/`allow-parse-error`/`block`/`override`/`skip-no-engine` with `scan_mode: "native-pre-commit"`. See spec 007's `plan.md` § *Approach* for the layered model and `tests/secrets-scan/` for the scenario suite.
