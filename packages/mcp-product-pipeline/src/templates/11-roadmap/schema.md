# Step 11 — Schema (required sections)

## Required sections

- horizon
- phases
- milestones
- risks
- v2-vision

## Recommended additional sections

- dependencies — sequencing constraints between phases or external dependencies (vendor onboarding, legal review, etc)
- buffer — buffer applied to estimates with rationale
- team — team shape assumed by the plan (solo / 2-3 / larger)

## Section content guidance

- **horizon** — total weeks/months to v1 launch + team shape recap.
- **phases** — 3-5 named phases (foundation / killer-flow / surrounding-features / polish-launch typical). Per phase: scope, week range, dependency notes.
- **milestones** — observable end-of-phase deliverables. 3-6 total. Each one is a "you'd recognise it when you see it" outcome.
- **risks** — per phase, the single biggest unknown + mitigation if any.
- **v2-vision** — 3-5 bullets sketching the next 3-6 months post-v1. Drives v1 platform/extension decisions.
