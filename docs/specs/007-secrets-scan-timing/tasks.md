## Notes

_Anything that came up during execution that doesn't belong in plan.md but is useful for the PR description or future readers._

- **The `gitleaks git --pre-commit --staged` invocation is the 2025-current shape**, replacing the deprecated `gitleaks protect --staged` that spec 006 shipped. Both layers migrate in this spec. Confirm against `gitleaks --version` (≥8.20 required) — older versions don't have the `git` subcommand and the native hook should fail-open with a clear message.
- **The override env-var bridge depends on Claude Code's `hookSpecificOutput.updatedInput` mechanism for PreToolUse hooks.** If that mechanism changes shape (it's documented but evolving), the override path breaks open — *not closed* — because the env var simply won't be set, so the native hook blocks normally. The fail-open posture aligns with the rest of the spec.
- **Test scripts under `tests/secrets-scan/` are pure POSIX shell** so the test suite can run in any environment that has `bash`, `git`, `jq`, and (for live-detection tests) `gitleaks`. No language runtime, consistent with the template philosophy.
- **The `core.hooksPath` Lazarus vector** means the install step (`git config core.hooksPath .githooks`) MUST be run manually by the fork developer. Do NOT add a `post-checkout` hook, a `git-init.sh` script, or any other automation that runs on clone — that pattern is the exact attack surface the 2025 Lazarus campaign exploited. The README per-fork checklist is the right place for the line; no auto-install.
- **AKIA test-vector reminder:** use `AKIA1234567890ABCDEF` for scenarios that need a detector trip (non-stopworded, pattern-valid, clearly fabricated). `AKIAIOSFODNN7EXAMPLE` is stopworded by gitleaks 8.21.2 and will NOT trip — this is documented in spec 006 § *Notes* and worth re-reading before writing fixtures.
