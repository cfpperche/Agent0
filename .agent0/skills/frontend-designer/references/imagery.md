# Imagery — optional `/image` generation (narrow, opt-in)

The artist MAY enrich a built surface with an on-brand image generated by the existing `/image` skill — but only as a **progressive enhancement** over a tracked neutral default. This is guidance over `/image`; it adds no new machinery. Decided in the decision-grade meeting `.agent0/meetings/frontend-designer-image-integration-*` (Claude + Codex, anchored ledger).

## When it applies

- **Modes:** `create` / `refine` only. **Never `explore`** (stays no-code, no image-gen).
- **Trigger:** the surface genuinely needs imagery (hero illustration, empty-state, decorative avatar, og-image) **and** the user opted in for that surface, **and** there is no suitable existing asset.
- **Default when any of those is false:** a tracked neutral placeholder (CSS/SVG/solid). The neutral path is what the surface *ships with*; generation is an override.

## The two hard limits

1. **draft tier only, auto.** `/image --tier=draft` (~$0.003, gitignored JPEG). `brand-text` / `brand-photo` are **never** invoked from `frontend-designer` — they cost more and produce *tracked brand assets*, which are `/product`'s brand phase. A real logo/wordmark/brand photo is referenced from the project or made via `/product` + standalone `/image`, never auto-generated here.
2. **Never a hard dep.** fal is paid + remote-only; the skill must work fully at zero key / zero credits / zero cost via the neutral default.

## The neutral fallback is the AUTHORED DEFAULT (structural, not convention)

Committed code renders a **tracked neutral media layer**. The generated draft is only a *preview override*, supplied through gitignored/local config — **never `import`ed as a build dependency** (a fresh clone wouldn't have the gitignored draft → broken build / broken visual).

Safest patterns:

- **Decorative background (preferred):** CSS two-layer where the neutral is independently present, so a missing generated URL can't break it:
  ```css
  .hero-art {
    background-image: var(--hero-generated, none), linear-gradient(135deg, var(--accent), var(--accent-2));
    background-size: cover;
  }
  /* --hero-generated is set ONLY by a gitignored/local override file when a draft exists */
  ```
  (CSS does NOT fall back after a 404 on a single `url()`; the neutral must be its own layer.)
- **Real `<img>` when required:** `src` defaults to a **tracked neutral asset**; the draft path comes from ignored/local config, with `onerror` swapping back to the neutral. Never a committed literal `src="assets/generated/mockups/…"`.

## Degradation ladder (founder directive — pre-check AND call-time)

1. **Off (default):** existing asset → tracked neutral placeholder.
2. **Opt-in → cheap pre-check:** `/image` helper exists, `FAL_KEY` set, shell tools present. If not → neutral, done.
3. **Attempt exactly ONE draft call.** Only the call reveals no-credits / bad key / `HTTP 402` / quota / provider failure / network / timeout.
4. **Classify any failure** (non-zero exec, `HTTP 402`, quota/payment text, curl timeout, malformed response, download failure) as **`image-fallback:<reason>`** — NOT a frontend failure → use the neutral layer. Build + contract still pass.
5. **Record** the outcome (generated path, or `image-fallback:<reason>`) in `design-direction.md` § Acceptance. **Redact** raw stderr first — never write `FAL_KEY`, auth headers, or provider payloads into a tracked doc.

`FAL_KEY` presence ≠ has-credits — the credit/quota state is only knowable at call-time, so the call-time catch (step 4) is mandatory, not just the pre-check.

## Cost discipline

- **Exactly one draft attempt per surface invocation, OUTSIDE the critique loop.** No try-N-prompts, no regenerate-on-vibes, no sub-agent fan-out.
- `/image` prints the cost before every call; surface it. The count cap + per-surface opt-in are the real guards for autonomous work.
- Any retry, multiple candidates, or a brand-tier call requires a **separate human instruction** or standalone `/image`.

## Done-proof interaction (UI acceptance)

- The generated image is **decorative** (`aria-hidden` / `alt=""`, or a real `alt` only if it conveys content). The project's UI test asserts the **semantic surface** (heading/controls/console), **never the bitmap**.
- **Verify with the draft disabled/missing too** — prove the neutral fallback renders and the UI test still passes, so a locally-successful draft never masks a broken neutral path.

## What is refused (no rule-of-three demand)

Provider discovery, credit-probing pre-checks, caching, budget dashboards, prompt-iteration spend loops, auto brand-asset generation, and asserting generated image *content* in the contract. If a real shell helper that invokes `/image` from `frontend-designer.sh` ever becomes warranted, write a light spec then (it becomes testable behavior, not guidance).
