# Step 6 — Schema (required sections)

## Required sections

- tokens
- components
- patterns
- accessibility-floor

## Recommended additional sections

- motion — durations and easing curves if motion is part of the brand
- iconography — icon style direction (outlined / filled / hand-drawn / etc) and primary icon source
- imagery — photo / illustration style if relevant

## Section content guidance

- **tokens** — color palette + semantic mapping, type scale (font, size, line-height, weight per step + semantic mapping), spacing scale, radius scale. Hex values for color, px/rem for type/spacing.
- **components** — list of components, with anatomy + states per component. Minimum set: Button, Input, Card, Modal/Sheet, Toast, NavBar. Add what the prototype actually needs.
- **patterns** — 4-6 named patterns showing how components combine (form layout, list-with-empty-state, error-handling, confirmation flow, etc).
- **accessibility-floor** — color contrast targets, focus state contract, keyboard navigation contract. At minimum WCAG AA.
